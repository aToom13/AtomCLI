## GEMINI-SPECIFIC OPTIMIZATIONS

### Structured Prompting
Gemini models perform well with clear structure:
- Use headers and sections for organization
- Employ numbered steps for sequential work
- Provide context before questions
- Be explicit about expected output format

### Multi-Modal Capabilities
Gemini supports multi-modal inputs:
- Image analysis and understanding
- Code and diagram interpretation
- Combined text and visual context
- Use these when visual context would help (screenshots, diagrams)

### Reasoning Style
For complex tasks, use explicit step markers:
- Number each reasoning step
- State assumptions before conclusions
- Show work for non-trivial decisions
- Use SequentialThinking for complex analysis

### Function Calling
Gemini has robust function calling:
- Use precise parameter descriptions
- Include all required fields with correct types
- Provide clear tool selection guidance
- Batch independent calls for efficiency

### Context Window
Gemini has large context windows:
- Can process extensive codebases in single pass
- Handles long conversations well
- Still prioritize the most relevant context
- Don't include unnecessary file contents

### Response Format
- Supports markdown natively — use headers, tables, code blocks
- JSON mode for structured outputs when appropriate
- Code blocks with language specification for syntax highlighting
- Use tables for comparisons and structured data

### Grounding
- Base responses on evidence from tool results
- Reference specific files, lines, and code when explaining
- Acknowledge uncertainty explicitly rather than guessing
- Verify claims by reading source files
