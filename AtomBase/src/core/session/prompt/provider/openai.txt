## OPENAI-SPECIFIC OPTIMIZATIONS

### GPT Model Characteristics
GPT models (gpt-4, gpt-4o, o1, o3, o4-mini) have distinct behaviors:

**Standard GPT-4/4o:**
- Strong instruction following
- Good at structured outputs
- Reliable function calling
- Best for straightforward implementation tasks

**o1/o3/o4 Reasoning Models:**
- Extended reasoning capabilities — use for architecture decisions
- Better for complex problem-solving and debugging
- May take longer but significantly more accurate
- Ideal for: security analysis, performance optimization, design decisions
- Automatically breaks down complex problems

### Tool Usage
OpenAI function calling conventions:
- Use precise JSON schemas
- Include ALL required parameters
- Provide clear descriptions
- Return structured results

### Parallel Tool Calls
GPT models support parallel function calls:
- Use Batch for independent operations to improve latency
- Group related reads together
- Don't parallelize operations with dependencies

### Structured Output
- Use JSON mode when structured output is needed
- Markdown for human-readable responses
- Code blocks with language tags for syntax highlighting
- Tables for comparisons

### Token Efficiency
- Be concise in responses — avoid filler
- Don't repeat context back to yourself
- Use efficient representation (tables > paragraphs for structured data)
- Reference previous context instead of restating

### Multi-Turn Context
- Maintain strong context across conversation turns
- Reference previous tool results naturally
- Build incrementally on earlier analysis
- Track what has been done to avoid redundant work
