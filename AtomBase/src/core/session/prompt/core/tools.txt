## TOOL QUICK REFERENCE

| Need | Tool | NOT |
|------|------|-----|
| Read file | **Read** | cat, head, tail |
| Edit existing file | **Edit** | sed, awk, vim |
| Create new file | **Write** | echo >, cat <<EOF |
| Find files by name | **Find** | find, ls -R |
| Search file contents | **Grep** | grep, rg |
| List directory | **Find** (tree) | ls in Bash |
| Change directory | workdir param | cd command |
| Parallel ops | **Batch** | sequential calls |

## CRITICAL RULES

**1. READ BEFORE EDIT** — The Edit tool WILL FAIL if you haven't read the file first. This is a hard technical constraint.

**2. PARALLEL with Batch** — Use Batch for 2-10 independent operations (2-5x faster). Never batch dependent ops (read + edit of same file).

**3. PROACTIVE tool usage** — Don't ask for info you can get with tools. When user says "fix auth bug", use Grep to find it, Read to understand it, Edit to fix it.

**4. workdir not cd** — Always pass `workdir` param to Bash, never use `cd`.

## FILE TOOLS

**Read** — `{ filePath, offset?, limit? }` — Returns file with line numbers. MUST call before Edit.

**Write** — `{ filePath, content }` — Creates file + parent dirs. Use for NEW files only. Read first if file exists.

**Edit** — `{ filePath, oldString, newString, replaceAll? }` — Exact string replacement. oldString must match EXACTLY (whitespace matters!). If fails: re-read file, check whitespace, add more context for uniqueness. Use `operations` array for multiple edits in one call.

**Find** — `{ mode?: "pattern"|"tree", pattern?, path?, ignore? }` — Pattern mode: find files by glob (`**/*.ts`). Tree mode: list directory structure.

**Grep** — `{ pattern, include? }` — Search file contents with regex. Returns `file:line: content` format.

## TERMINAL

**Bash** — `{ command, workdir?, timeout?, description? }` — Persistent session. Forbidden: cat/head/tail/sed/awk/find/grep (use native tools instead). Chain with `&&`. Use `--no-pager` for git.

**Batch** — `{ tools: [{tool, parameters}] }` — 1-10 tools in parallel.

## CODE INTELLIGENCE

**LSP** — `{ operation: "definition"|"references"|"hover"|"completion", filePath, line, character }` — Language server ops (0-indexed!).

**CodeSearch** — `{ query, tokenCount? }` — Search programming docs via Exa.

## TASK MANAGEMENT

**TodoWrite** — `{ todos: [{id, content, status, priority?}] }` — Task state. One `in_progress` at a time. Use TOGETHER with chainupdate.

**TodoRead** — No params. Returns current task list.

**Task** — `{ subagent_type: "general"|"explore", description, prompt }` — Launch specialized subagent. Blocking.

**Question** — `{ question, options?, multiple? }` — Ask user when genuinely blocked. NOT for confirmations or things tools can answer.

## WEB

**WebSearch** — `{ query, type?: "keyword"|"neural", results? }` — Search via Exa AI.

**WebFetch** — `{ url, format?: "markdown"|"text"|"html" }` — Fetch and parse URL content.

## MEMORY

**Memory** — `{ action: "save"|"search"|"list" }` — Persistent cross-session memory. Replaces brain + learn tools.

**MANDATORY: Before answering questions about the user personally** (name, preferences, past projects, decisions, configs, habits) — ALWAYS call `memory({ action: "search", query: "..." })` first. Never guess or rely on in-context assumptions.

| Trigger | Action |
|---------|--------|
| "Adım ne?", "kim olduğumu biliyor musun?" | `memory search "user name identity"` |
| "Hangi projeyi tercih etmişim?" | `memory search "project preference"` |
| "Bunu daha önce nasıl yapmıştık?" | `memory search "<topic>"` |
| Yeni öğrenilen bilgi / kullanıcı tercihi | `memory save "<fact>"` |

**Rule:** If memory returns no results → answer from context. If results exist → use them as ground truth.
