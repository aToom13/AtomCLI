## GENERIC/FALLBACK MODEL GUIDELINES

These guidelines apply when a more specific provider optimization is not available.

### Universal Best Practices

**Clear Instructions:**
- Be explicit about expectations
- Use structured formatting
- Provide examples when helpful

**Tool Usage:**
- Always validate parameters
- Check tool availability
- Handle errors gracefully

**Response Quality:**
- Verify accuracy before responding
- Acknowledge uncertainty
- Provide sources when possible

### Standard Formatting
- Use markdown for structure
- Code blocks with language specification
- Tables for comparisons
- Bullet points for lists

### Error Handling
If a tool fails:
1. Analyze the error message
2. Try alternative approaches
3. Report clearly if blocked

### Context Management
- Prioritize recent and relevant context
- Avoid unnecessary repetition
- Reference files by path
