Executes bash commands in a persistent shell session with optional timeout.

Commands run in ${directory} by default. Use `workdir` for different directories — AVOID `cd <dir> && <cmd>`.

IMPORTANT: Do NOT use bash for file operations. Use dedicated tools instead:
- File search: Find (not find/ls) | Content search: Grep (not grep/rg)
- Read: Read (not cat/head/tail) | Edit: Edit (not sed/awk) | Write: Write (not echo >)

Rules:
- Always quote file paths with spaces: `mkdir "/path with spaces/"` (correct)
- Output over ${maxLines} lines / ${maxBytes} bytes is auto-truncated; full output saved to file. No need for head/tail.
- Independent commands → multiple parallel Bash calls in one message. Dependent → chain with `&&`.
- Use `;` only when sequential but failure of earlier commands is acceptable.
- DO NOT use newlines to separate commands (newlines ok in quoted strings).
- Timeout: 120000ms default. Override via timeout parameter.
- Write a clear 5-10 word description of what each command does.

# Directory verification
Before creating new directories/files, use `ls` to verify the parent exists and is correct.

# Multiple commands
When issuing multiple commands:
- Independent + likely to succeed → parallel Bash calls
- Dependent (mkdir before cp, git add before commit) → single call with `&&`

# Git safety (see skill:git-commit for the full workflow)
- NEVER update git config, run destructive commands (push --force, hard reset), or skip hooks unless explicitly asked
- NEVER commit unless explicitly asked. NEVER use interactive flags (-i)
- NEVER use git commit --amend unless: user requested + HEAD is yours + not pushed
- If commit fails, create NEW commit — never amend a failed commit
- Do NOT push unless explicitly asked
- For creating PRs, use the skill:github-pr workflow

# Other
- Use `gh` for all GitHub operations (issues, PRs, checks, releases)
- View PR comments: `gh api repos/owner/repo/pulls/123/comments`
