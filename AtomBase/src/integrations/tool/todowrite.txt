Create and manage a structured task list for the current session. Tracks progress and shows the user what's happening.

## When to use
- Tasks requiring 3+ distinct steps
- User provides multiple tasks or explicitly requests a todo list
- After receiving new instructions — capture requirements immediately
- After completing a task — mark complete, add follow-ups
- When starting work on a task — mark as in_progress

## When NOT to use
- Single straightforward task completable in <3 steps
- Purely conversational or informational requests
- Trivial tasks with no tracking benefit

NOTE: If there is only one trivial task, just do it directly without creating a todo list.

## Task states
- pending: Not started
- in_progress: Currently working (ONE at a time)
- completed: Finished
- cancelled: No longer needed

## Rules
- Update task status in real-time as you work
- Mark tasks complete IMMEDIATELY after finishing (don't batch completions)
- Only ONE task in_progress at any time
- Complete current tasks before starting new ones
- Cancel tasks that become irrelevant
- Create specific, actionable items with clear, descriptive names
- Break complex tasks into smaller manageable steps

When in doubt, use this tool — proactive tracking demonstrates thoroughness and ensures all requirements are met.
